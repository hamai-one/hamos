1. **Scraping Trends (Background Process)** -> K-Means clustering untuk menemukan pola tren dominan.
2. **Content Gen Request** -> Prompt diserahkan ke `deep-research-max` untuk skrip, dan `GENERATOR_STUDIO` untuk visual.
3. **Scheduling Output** -> Disimpan ke dalam SQLite Database (OPFS) `massiveDb.worker.ts` untuk queue management.
4. **Auto-Posting Execution** -> CRON-job mem-fire API saat *prime time* prediksi audiens.
5. **Analytics Review & Feedback Loop** -> Pembacaan webhook 24 jam. Jika performa > 80% threshold, strategi di-commit ke `Hamli Core Memory` untuk post selanjutnya.
